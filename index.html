CarbonWise AI System Topology

To scale CarbonWise AI to millions of users, the platform leverages a fully decoupled serverless architecture connected with high-speed caching networks and secure edge workers.

1
Client UI

Highly responsive SPA client built using standards-compliant HTML5/CSS3. Optimized for low-bandwidth environments.

2
Edge Delivery Network

Cloud-native deployment using Vercel/AWS with CDN edge delivery caches static templates near the user, improving latency.

3
Fast API Gateway

Handles authorization, routes log calculations to localized edge nodes, and queues larger receipt/utility bill analysis jobs.

4
AI & OCR Worker Engine

Utilizes lightweight PyTorch models for trend forecasting, LayoutLM neural networks for invoice structuring, and LLMs for explaining coach suggestions.

Relational PostgreSQL Schema

The database schema is optimized for write-heavy logging streams and fast retrieval of community leaderboards and streaks.

-- Users Table
CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    username VARCHAR(50) UNIQUE NOT NULL,
    email VARCHAR(100) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    sustainability_score INT DEFAULT 100 CHECK (sustainability_score BETWEEN 0 AND 100),
    current_streak INT DEFAULT 0,
    total_xp INT DEFAULT 0,
    monthly_budget_kg NUMERIC DEFAULT 150.0
);

-- Daily/Weekly Emission Logs
CREATE TABLE emission_logs (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id) ON DELETE CASCADE,
    logged_date DATE NOT NULL,
    category VARCHAR(50) NOT NULL, -- 'transport', 'energy', 'food', 'shopping', 'waste', 'water', 'digital'
    value_input NUMERIC NOT NULL,  -- e.g., miles traveled, kWh consumed
    unit VARCHAR(20) NOT NULL,     -- e.g., 'km', 'kWh', 'kg'
    co2e_kg NUMERIC NOT NULL,      -- Calculated emission value
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

-- Gamification Challenges & Badges
CREATE TABLE badges (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name VARCHAR(50) UNIQUE NOT NULL,
    description TEXT NOT NULL,
    icon_url VARCHAR(255) NOT NULL,
    xp_reward INT DEFAULT 100
);

CREATE TABLE user_badges (
    user_id UUID REFERENCES users(id) ON DELETE CASCADE,
    badge_id UUID REFERENCES badges(id) ON DELETE CASCADE,
    earned_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    PRIMARY KEY (user_id, badge_id)
);

-- AI Generated Action Plans
CREATE TABLE action_plans (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID REFERENCES users(id) ON DELETE CASCADE,
    action_description TEXT NOT NULL,
    why_suggested TEXT NOT NULL,
    scientific_reasoning TEXT NOT NULL,
    impact_level VARCHAR(10) CHECK (impact_level IN ('High', 'Medium', 'Low')),
    effort_level VARCHAR(10) CHECK (effort_level IN ('High', 'Medium', 'Low')),
    co2e_reduction_kg NUMERIC NOT NULL,
    status VARCHAR(20) DEFAULT 'pending' CHECK (status IN ('pending', 'completed', 'skipped')),
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

Responsible AI & Trust

End-to-End Encryption

All client logs, uploaded billing documents, and coach dialogues are encrypted in transit using TLS 1.3 and at rest with AES-256 GCM key wrappers.

GDPR Compliant Architecture

We enforce data minimisation principles. No tracking trackers or marketing cookies are injected. Users retain a one-click purge action to scrub their lifecycle footprint history.

Transparent Decision Making

Calculation values align with scientific standard coefficients. We expose our mathematical formulas: e.g. for Gasoline Cars, the emission is calculated exactly as: km * 0.18 kg CO₂e/km.

Explainable Recommendations

Every automated suggestion comes with an associated impact level, an estimated carbon savings, and references to environmental research to verify efficiency.
